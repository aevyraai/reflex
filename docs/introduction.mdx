aevyra-reflex is an agentic prompt optimizer. Give it a dataset, a system prompt,
and a target score — it diagnoses why your model is underperforming, reasons about
the failure patterns, and rewrites the prompt until the scores match your target.
All evaluation runs through [aevyra-verdict](https://github.com/aevyraai/verdict).

## What it does

Given a dataset and a starting prompt, aevyra-reflex:

1. Sets a target score — from a verdict benchmark, a live model comparison, or an explicit threshold
2. Runs a baseline eval with verdict to measure the starting score
3. Analyzes failure patterns and rewrites the prompt to fix them
4. Re-evaluates after each rewrite, iterating until the score meets the target
5. Returns a before/after comparison with the optimized prompt

## When to use it

- You ran verdict and model A beats model B — you want to close the gap through prompt engineering, not by switching models
- A model scores poorly on your eval — you want a better prompt, not a bigger model
- You're iterating on a system prompt and want to automate the feedback loop
- You want to understand *why* a prompt works (the analysis teaches prompt engineering)

## Optimization strategies

The **auto** strategy (default) picks the right technique for each phase.
You can also run any strategy directly.

```
                ┌──────────────┐
                │   Dataset    │
                │  (JSONL)     │
                └──────┬───────┘
                       │
                       ▼
  ┌─────────────────────────────────────┐
  │         aevyra-reflex               │
  │                                     │
  │  ┌───────────┐   ┌──────────────┐  │
  │  │  Strategy  │──▶│  Reasoning   │  │

  │  └─────┬─────┘   └──────────────┘  │
  │        │                            │
  │        ▼                            │
  │  ┌───────────┐                      │
```
