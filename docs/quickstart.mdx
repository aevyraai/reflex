## Install

```bash
pip install aevyra-reflex
```

This also installs [aevyra-verdict](https://github.com/aevyraai/verdict) for
evaluation.

## Set your API keys

By default, reflex uses Claude for reasoning (analyzing failures, rewriting
prompts). You also need a key for whatever model you're optimizing:

```bash
export ANTHROPIC_API_KEY=sk-ant-...    # default reasoning model (or use --reasoning-model for local)
export OPENAI_API_KEY=sk-...           # if optimizing an OpenAI model
```

For local models (Ollama), no extra key is needed. You can also use a local
reasoning model — see [Choosing a reasoning model](/docs/configuration#choosing-a-reasoning-model).

## Prepare a dataset

Use the same JSONL format as verdict — each line has messages and an ideal answer:

```json
{"messages": [{"role": "user", "content": "What is the capital of France?"}], "ideal": "Paris"}
{"messages": [{"role": "user", "content": "Explain binary search in one sentence."}], "ideal": "Binary search repeatedly halves a sorted array to find a target value in O(log n) time."}
```

## Write a starting prompt

Create a plain text file with your system prompt:

```markdown
You are a helpful assistant. Answer questions concisely.
```

This is the prompt reflex will improve.

## Run your first optimization

```bash
aevyra-reflex optimize dataset.jsonl prompt.md \
  -m local/llama3.1 \
  -o best_prompt.md
```

You'll see a baseline eval, optimization iterations with rising scores, and a
final verification:

## Use the Python API

```python
from aevyra_verdict import Dataset, RougeScore
from aevyra_reflex import PromptOptimizer, OptimizerConfig

dataset = Dataset.from_jsonl("dataset.jsonl")

config = OptimizerConfig(
    strategy="auto",
    max_iterations=10,
    score_threshold=0.85,
)

result = (
    PromptOptimizer(config)
    .set_dataset(dataset)
    .add_provider("local", "llama3.1")
    .add_metric(RougeScore())
    .run("You are a helpful assistant.")
)

print(result.summary())
result.save_best_prompt("best_prompt.md")
```

## Set a real target (verdict → reflex)

Instead of an arbitrary threshold, set the target from a real benchmark. If you
already ran `aevyra-verdict`, pass the results file:

```bash
# Use gpt-4o-mini's score as the target for llama
aevyra-reflex optimize dataset.jsonl prompt.md \
  -m local/llama3.1 \
  --verdict-results results.json \
  -o best_prompt.md
```

Or let reflex benchmark for you in one command:

```bash
# Benchmark llama vs gpt-4o-mini, then optimize llama to match
aevyra-reflex optimize dataset.jsonl prompt.md \
  -m local/llama3.1 \
  --target openai/gpt-4o-mini \
  -o best_prompt.md
```

## Use a cloud provider

```bash
# OpenRouter (pay-per-token, many models)
aevyra-reflex optimize dataset.jsonl prompt.md \
  -m openrouter/meta-llama/llama-3.1-8b-instruct \
  --target openai/gpt-4o-mini \
  -o best_prompt.md

# Together AI
aevyra-reflex optimize dataset.jsonl prompt.md \
  -m together/meta-llama/Llama-3.1-8B-Instruct \
  -o best_prompt.md
```
